## 1. Project Context
The "SmartBacklog" project aims to resolve the "blank page syndrome" often faced by Product Owners when creating Jira/Trello tickets. By integrating an AI layer, the application automates repetitive tasks such as writing acceptance criteria, estimating complexity, and defining priorities.

## 2. Methodology Applied: Agile Scrum
Following the principles taught in the IS Modeling course, this project strictly adheres to the Agile methodology:
*   **Iterative Delivery:** The project is split into two distinct Sprints.
*   **Prioritization:** The backlog is managed using the MoSCoW method to ensure value delivery.
*   **Estimation:** Tasks are estimated using Story Points based on the Fibonacci sequence.
*   **Continuous Improvement:** A Sprint Review and Retrospective will be documented after Sprint 1.
